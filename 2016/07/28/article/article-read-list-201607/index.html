Here’s How Pokemon GO Became An Overnight Phenomenon

Article Link

  • Pokemon Go 已經開始被犯罪者拿來引誘受害者的工具
  • 2008年 Nintendo 宣布不開發手遊，讓股價一落千丈，就算現在 Pokemon Go 使股價大漲35%仍不及2008年的1/3
  • Pokemon Go 是由 Niantic Labs 跟 Google 一同針對 Reality Mobile Games 所開發出來
  • Pokemon 是由1995年 Satoshi Tajiri 這位童年的昆蟲收集狂所想出來的概念，至今已有19季動畫、18部電影、卡牌遊戲跟無數的玩具
  • Pokemon Go 火紅的原因可以歸類為以下三點
    • 懷舊
    • 真正的移動遊戲
    • 病毒式的擴散（無人機作弊太好笑了）
  • 存在一些 Server 問題
    • 只推出少數的國家
    • 隱私
    • GPS 追蹤
    • 電池 life
    • 登入？
    • 加入其它玩法

The Worst Server Setup Mistake You Can Make

Article Link

  • 將 server 的時區設定為 UTC
  • DST 會有消失的1小時問題
  • UTC 沒有日光節約時間、是普遍的且成熟的組織大多用UTC

How To Safely Store A Password

Article Link

  • MD5 的編碼使用一般的 Server 針對6字元大約只需40秒就可以破解
  • 加 salt 對字典及暴力破解並無幫助，因為 password hash 及 salt 都是從 database 取出
  • BCrypt 可以利用 work factor 來決定 hash 的複雜度
  • 針對 work factor 5 的等級，每40秒破解一次的話要12年才可以成功
  • Rainbow Table 依舊是無法有效的防護

How Humble Bundle stops online fraud

Article Link

  • Humble Bundle 是跟 Steam 很像的遊戲販售平台
  • 為了防止 fraud 依循以下幾個步驟
    1. 風險評估 - 透過 Sift Secience 的 Data Mining 可以有效的找出可疑的交易
    2. SMS 驗證 - 如果該交易是高風險，就會請客戶透過手機驗證 SMS
    3. 人工 Review - 如果該交易仍然屬高風險，就會依各交易記錄、歷史資料人工決定是否為合法，但通常還是會放行，畢竟誤鎖合法用戶比被詐騙還糟
    4. 限速 & 驗證碼 - 如果上述都失敗時會透過限速及驗證碼來降低傷害（例子看沒有懂）
    5. Key 消除 - 如果發現交易是很罕見（這邊應該是指假 Key ），就會取消訂單重發 Key 給開發者
    6. 持續觀查 - 跟其它的平台互助一起診斷、除錯和改善系統
  • 對客戶而言 - 如果被誤鎖，並不是故意的，也請耐心等待結果，這大部份發生在短期大金額的交易
  • 對開發者而言 - 已經花了很多年在解決 fraud 的問題，所以可以相信 Humble 的交易安全
